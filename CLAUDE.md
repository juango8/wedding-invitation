# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Single-page wedding invitation site (Juango & Mafer, July 18 2026, Arequipa, Perú) plus an admin panel for the couple, backed by Supabase. Vite + React 19 + TypeScript + Tailwind CSS 3. All user-facing text is in Spanish — including the admin panel.

History: the site originally posted RSVPs to a Google Sheet via Apps Script; the migration to Supabase is documented in [docs/RESTRUCTURE_PLAN.md](docs/RESTRUCTURE_PLAN.md) (kept as the migration record — this file describes the current state).

## Commands

```
npm run dev         # dev server (serves under /wedding-invitation/)
npm run build       # tsc -b, then vite build (output: dist/)
npm run lint        # eslint .
npm run preview     # preview the production build
npm run db:generate # drizzle-kit: diff db/schema.ts into a migration
npm run db:migrate  # drizzle-kit: apply db/migrations to DATABASE_URL
npm run db:studio   # drizzle-kit studio
npm run db:import -- archivo.csv [--dry]  # bulk import/update guests from CSV
```

There are no tests.

Note: the dev/build/preview scripts invoke Vite through `node -r ./crypto-polyfill.cjs` (polyfills `crypto.getRandomValues` for old Node). Don't simplify them to plain `vite` calls.

## Architecture

- **Two pages via hash routing** ([useHashRoute.ts](src/hooks/useHashRoute.ts), no router lib): the invitation (default) and the admin panel at `#/manage`. [App.tsx](src/App.tsx) switches between them. Section anchors (`#our-story`, `#venue`, `#itinerary`, `#rsvp`, `#registry`) are unaffected by the routing; [Sidebar.tsx](src/components/Sidebar.tsx) navigates with them and new sections need both the `id` and a `NAV_ITEMS` entry. `RSVPSection` also contains the Registry section and the site footer, despite its name.
- **Guest personalization**: invitations are per-guest links `?g=<token>` (10-char unambiguous codes generated by a DB trigger). [useGuest.ts](src/hooks/useGuest.ts) resolves the token with states `none` / `loading` / `found` / `invalid` / `error` — `invalid` (unknown token) and `error` (network/8s timeout, offers retry) deliberately get different UX. There is **no free-text RSVP**: without a valid link the form is replaced by a notice + WhatsApp button (number in [contact.ts](src/lib/contact.ts)). Guests can change their answer by reopening their link.
- **Backend = Supabase, no server of our own.** All runtime access goes through `supabase-js` ([supabase.ts](src/lib/supabase.ts) — the client is `null` when env vars are missing and every caller degrades gracefully, same pattern as VenueMap with a missing Maps key).
  - **Schema source of truth**: [db/schema.ts](db/schema.ts) (Drizzle). Migrations live in [db/migrations/](db/migrations) — generated ones plus hand-written SQL (RLS, RPCs, triggers; create those with `npx drizzle-kit generate --custom`). drizzle-kit connects via `DATABASE_URL` (session pooler, port 5432 — the direct connection is IPv6-only).
  - **Public flow**: the anon key can only execute two SECURITY DEFINER RPCs — `get_guest(p_token)` and `submit_rsvp(p_token, p_attending, p_message)` ([api.ts](src/lib/api.ts)). Zero table access for anon → the guest list cannot be enumerated. `submit_rsvp` caps messages at 500 chars and sets `responded_at`.
  - **Admin flow**: Supabase Auth with Google (PKCE flow, so the OAuth callback `?code=` coexists with hash routing). Authorization is the `admins` table enforced **in RLS**; the `is_admin()` RPC only gates the UI ([useAdminSession.ts](src/hooks/useAdminSession.ts)). Add an admin: `insert into public.admins (email) values ('x@gmail.com');` in the SQL editor (lowercase).
- **Manage panel** ([src/manage/](src/manage)): `GuestPanel` (stats, accent-insensitive search, status/group filters, sortable table → cards on mobile, CSV export, manual status override for WhatsApp replies), `GuestFormModal` (add/edit), `useGuests` (direct table access under RLS; mutations refetch), and [messages.ts](src/manage/messages.ts) — invite-message templates plus `SITE_URL`: copied links always point at prod, even from localhost.
- **Map**: [VenueMap.tsx](src/components/VenueMap.tsx) loads the Google Maps JS API dynamically using `VITE_GOOGLE_MAPS_API_KEY`; missing key renders a placeholder instead of failing. Venue coordinates and custom map styles live in that file.
- **Styling**: custom Tailwind palette in [tailwind.config.ts](tailwind.config.ts) — note the `rose` color group is actually brown/green earth tones (`rose` = brown #6c402a, `rose-dark` = green #3a4a31), and `warm-*` for text/backgrounds. Shared utility classes `.section-title` and `.nav-link` are defined in [src/index.css](src/index.css). Serif = Playfair Display, sans = Inter.
- [cloudinary-setup.mjs](cloudinary-setup.mjs) is a standalone one-off script (run with `node`) to test Cloudinary uploads; it is not part of the site build.

## Environment & deployment

- Local values go in `.env` (gitignored; see [.env.example](.env.example)):
  - `VITE_SUPABASE_URL` + `VITE_SUPABASE_ANON_KEY` — public by design (baked into the bundle, protected by RLS). Must also exist as GitHub Actions secrets with the same names.
  - `DATABASE_URL` — **secret, local-only** (Postgres password inside; URL-encode special chars, e.g. `@` → `%40`). Used by drizzle-kit and `db:import`. Never a `VITE_` var, never in CI.
  - `VITE_GOOGLE_MAPS_API_KEY` — also a GitHub Actions secret.
- Deploys to GitHub Pages via [deploy.yml](.github/workflows/deploy.yml) on every push to `main`. The Vite `base` is `/wedding-invitation/` ([vite.config.ts](vite.config.ts)) — asset paths and OAuth `redirectTo` assume that subpath.
- [keepalive.yml](.github/workflows/keepalive.yml) pings a Supabase RPC twice a week — **free-tier projects pause after ~7 idle days**, which would take the live site down.
- If the Supabase project is ever recreated: enable the **Data API** (`public` schema exposed) — without it REST returns `PGRST002`; configure the Google provider; register redirect URLs for `http://localhost:5173/**`, `http://localhost:4173/**` and the prod URL.
